Assembly Language

The First Step In Making Software Development Easier And More Efficient Was The Creation Of Assembly Language.
It Is The One Level Up From Machine Language.
However, It Is Classified As A Low-Level Language Because Detailed Knowledge Of Hardware Is Still Required.

  1. In Assembly Language, Programs Are Written In Combination Of Mnemonic Codes And Binary Codes.
  2. Mnemonic Codes Are Short Words That Represent A Complete Instruction.
    For example, LDA for Load, TRAN For Translation, ADD For Addition, SUB For Subtraction & MUL For Multiplication.
  3. These Mnemonic Codes Could Be Typed And Read Easily By A Programmer.
  4. To Run On A Computer, Assembly Language Much Be Converted Into A Machine Language.
  5. An Assembler Converts The Assembly Language Statements Into Machine Language.
  6. Assembly Language Is Easier To Understand Than Machine Language.
  7. In Comparision To Machine Language, It Is Easier To Debug Errors In Programs.
  8. Assembly Language Is Machine Dependent.
    A Program Written For One Computer Might Not Run In Other Computers With Different Hardware Configurations.
